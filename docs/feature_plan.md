# 🚀 AI 감정 일기 고도화 작업 계획서 (Feature Roadmap)

AI 감정 일기의 사용자 경험을 극대화하기 위한 3단계 기능 확장 계획입니다.

---

## 📅 1단계: 일기 저장 및 관리 (LocalStorage)
**목표**: 사용자가 작성한 일기와 AI의 분석 결과를 로컬에 보존하여 나만의 감정 기록장을 구축합니다.

### 1-1. 데이터 구조 설계
```json
{
  "diaries": [
    {
      "id": "unique_timestamp",
      "date": "2026-04-09",
      "content": "오늘의 일기 내용...",
      "emotion": "기쁨",
      "aiResponse": "AI의 응원 메시지...",
      "timestamp": 1712644200000
    }
  ]
}
```

### 1-2. 주요 구현 사항
- **CRUD 로직**: 
  - 생성: '분석 완료' 시점에 LocalStorage에 새 일기 객체 추가.
  - 조회: 앱 로드 시 저장된 일기 목록을 불러와 하단에 리스트 형태로 출력.
  - 삭제: 특정 일기 기록을 삭제할 수 있는 기능 추가.
- **UI 업데이트**: 
  - 메인 화면 하단에 '나의 지난 기록' 섹션 추가 (유리 모티프 카드 디자인 적용).

---

## 🎨 2단계: 프리미엄 다크 모드 (Dark Mode)
**목표**: 야간에도 눈의 피로 없이 감성적으로 일기를 작성할 수 있는 환경을 제공합니다.

### 2-1. 디자인 시스템 구축
- **Light Theme**: 기존의 화이트/라벤더 글래스모피즘 유지.
- **Dark Theme**: 심야 분위기의 딥 네이비(#0f172a) 및 다크 슬레이트 배경, 네온 퍼플 포인트.

### 2-2. 주요 구현 사항
- **테마 스위처**: 우측 상단에 해/달 아이콘 토글 버튼 추가.
- **자동 감지**: 사용자의 OS 테마 설정(`prefers-color-scheme`)에 따른 자동 초기화.
- **상태 유지**: 선택한 테마 설정을 LocalStorage에 저장하여 재방문 시 유지.

---

## 📊 3단계: 감정 통계 시각화 (Emotion Statistics)
**목표**: 수집된 데이터를 바탕으로 사용자의 감정 흐름을 한눈에 파악할 수 있는 인사이트를 제공합니다.

### 3-1. 통계 데이터 분석
- 누적된 일기 데이터에서 감정 키워드(기쁨, 슬픔 등) 빈도수 추출.
- 일주일 단위의 감정 변화 추이 계산.

### 3-2. 주요 구현 사항
- **차트 라이브러리 연동**: `Chart.js` 또는 가벼운 CSS 기반 막대 차트 사용.
- **통계 대시보드**: 
  - '이번 주 감정 분포' (도넛 차트).
  - '최근 7일 감정 변화 흐름' (라인 차트).
- **인터랙션**: 차트 요소 클릭 시 해당 날짜의 일기로 바로 이동하는 편의성 제공.

---

## 🛠️ 개발 환경 및 도구
- **Storage**: Browser LocalStorage API
- **Iconography**: Lucide-react (기존 유지)
- **Charts**: Chart.js (CDN 또는 NPM)
- **CSS**: Vanilla CSS Variable 기반 테마 관리

---

## 📝 향후 일정 (Timeline)
1. **Day 1**: LocalStorage 로직 구현 및 지난 기록 UI 완성.
2. **Day 2**: 다크 모드 색상 토큰 정의 및 토글 기능 구현.
3. **Day 3**: 데이터 통계 엔진 구축 및 차트 시각화 완료.
